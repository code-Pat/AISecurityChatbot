[JWT]
JWT(JSON Web Token)는 두 개체 사이에 정보를 안전하게 전송하기 위한 개방형 표준(RFC 7519)이다.
헤더, 페이로드, 서명 세 부분으로 구성되며 Base64로 인코딩된다.
주로 인증과 정보 교환에 사용된다.
서명을 통해 토큰의 무결성을 검증할 수 있다.

[OAuth 2.0]
OAuth 2.0은 제3자 애플리케이션이 사용자 대신 서비스에 접근할 수 있도록 권한을 위임하는 프로토콜이다.
Access Token과 Refresh Token을 사용한다.
Authorization Code, Client Credentials 등 여러 Grant Type이 있다.
Bearer Token 방식으로 API 요청 시 인증한다.

[OTP]
OTP(One-Time Password)는 한 번만 사용할 수 있는 일회용 비밀번호다.
TOTP(Time-based OTP)는 시간을 기반으로 생성되며 Google Authenticator가 대표적이다.
HOTP(HMAC-based OTP)는 카운터 기반으로 생성된다.
주로 2단계 인증(2FA)에 활용된다.

[FIDO]
FIDO(Fast Identity Online)는 비밀번호 없는 인증을 위한 표준이다.
공개키 암호화 방식을 사용해 피싱 공격에 강하다.
FIDO2는 WebAuthn과 CTAP으로 구성된다.
생체인식, 보안키 등 다양한 인증 수단을 지원한다.
